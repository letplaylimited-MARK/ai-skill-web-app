01  ·  VALUE SYSTEM

价值体系

AI Skill 编排系统通过结构化的六层Skill流水线，将混沌的AI工作流转变为可重复、可审计、可迭代的工程化流程。

价值金字塔

🎯 战略层：决策加速
⚙️ 执行层：流程自动化
🔧 工具层：六层Skill编排
📦 基础层：Next.js + Prisma + PostgreSQL
🚀

效率提升 10x

将原本需要数小时手工整理的需求分析、技术选型、执行规划，压缩到分钟级完成。六层Skill协同消除重复劳动。

🎓

知识沉淀

每次执行结果自动保存为结构化JSON和Markdown，形成可检索的组织知识库。历史执行记录支持对比和复用。

🔄

五种执行模式适配不同场景。模式E深度精炼支持多轮迭代，每次基于上次输出持续优化，逼近最优解。

05  ·  INSTALLATION

如何安装

5步完成本地环境安装，从 clone 到运行约需 5 分钟（需预装 Node.js 20+ 和 PostgreSQL 15+）。

1

克隆仓库

从 GitHub 拉取源码

# 克隆仓库
git clone https://github.com/letplaylimited-MARK/ai-skill-web-app.git
cd ai-skill-web-app
2

安装依赖

使用 pnpm 安装所有依赖包（推荐）或 npm

# 使用 pnpm（推荐）
pnpm install

# 或使用 npm
npm install
3

配置环境变量

复制示例配置文件并填写 API Key 和数据库连接

cp .env.example .env.local

# 编辑 .env.local，填写以下变量：
DATABASE_URL="postgresql://user:password@localhost:5432/aiskill"
ANTHROPIC_API_KEY="sk-ant-api03-..."
OPENAI_API_KEY="sk-proj-..."        # 可选
NEXTAUTH_SECRET="your-random-secret-32-chars"
NEXTAUTH_URL="http://localhost:3000"
4

初始化数据库

运行 Prisma 迁移并加载种子数据

# 运行数据库迁移
pnpm prisma migrate dev --name init

# 加载示例数据（可选）
pnpm prisma db seed

# 查看数据库管理界面
pnpm prisma studio
5

启动开发服务器

启动 Next.js 开发服务器，访问 http://localhost:3000

# 开发模式（热重载）
pnpm dev

# 构建生产版本
pnpm build && pnpm start

# 访问应用
# 浏览器打开: http://localhost:3000
系统要求

Node.js 20.x+, pnpm 9.x+, PostgreSQL 15+, Anthropic API Key（必需），OpenAI API Key（可选）

06  ·  DEPLOYMENT

如何部署

支持三种部署方式，从零代码的 Vercel 一键部署到完全自控的 VPS 自托管。

Vercel 部署（推荐新手）

1

Fork 仓库

将 GitHub 仓库 Fork 到自己账号下

2

连接 Vercel

访问 vercel.com/new，选择 Import Git Repository，选中 fork 的仓库

3

配置环境变量

在 Vercel Project Settings → Environment Variables 添加以下变量

DATABASE_URL="postgresql://..."     # Neon/Supabase/Railway
ANTHROPIC_API_KEY="sk-ant-api03-..."
NEXTAUTH_SECRET="$(openssl rand -base64 32)"
NEXTAUTH_URL="https://your-app.vercel.app"
4

部署

点击 Deploy，等待约 2 分钟，自动获得 HTTPS 域名

推荐搭配数据库： Neon（免费套餐），Supabase（PostgreSQL托管），PlanetScale（MySQL兼容）

Docker Compose 部署

# docker-compose.yml
version: '3.9'
services:
  app:
    build: .
    ports: ["3000:3000"]
    environment:
      - DATABASE_URL=postgresql://postgres:pass@db:5432/aiskill
      - ANTHROPIC_API_KEY=${ANTHROPIC_API_KEY}
      - NEXTAUTH_SECRET=${NEXTAUTH_SECRET}
      - NEXTAUTH_URL=http://localhost:3000
    depends_on: [db]
  db:
    image: postgres:15-alpine
    environment:
      POSTGRES_PASSWORD: pass
      POSTGRES_DB: aiskill
    volumes: [pgdata:/var/lib/postgresql/data]
volumes:
  pgdata:
# 启动所有服务
docker compose up -d

# 运行数据库迁移
docker compose exec app pnpm prisma migrate deploy

# 查看日志
docker compose logs -f app

07  ·  SKILLS

六层 Skill 详解

每个 Skill 承担独立职责，输出作为下一层的输入，形成端到端的智能流水线。

S00
🧭
路由导航官
Navigator · 意图识别 & 路径决策

解构用户需求，分析意图置信度（≥80%直接路由/60-79%确认/<60%追问），精准路由到后续Skill，输出标准化交接包YAML。

意图识别 置信度评分 路径决策
INPUT用户自然语言需求描述（任意格式） OUTPUT意图分析报告 + YAML交接包 + 推荐路由路径
S01
✍️
提示词工程师
Prompt Engineer · 四维框架优化

使用 ROLE/TASK/CONTEXT/FORMAT 四维框架对原始Prompt进行结构化重构和优化，提升AI响应质量和一致性。

ROLE定义 TASK明确 FORMAT规范
INPUTS00交接包 + 原始需求文本 OUTPUT优化后的结构化Prompt + 使用说明 + 评估分数
S02
📋
SOP工程师
SOP Engineer · 10阶段标准化流程

通过10阶段标准流程（需求→设计→原型→评审→开发→测试→文档→部署→验收→迭代），生成可复用的Skill工程包。

10阶段流程 阶段门控 可复用包
INPUTS01优化Prompt + 项目范围说明 OUTPUT完整SOP文档 + Skill工程包 + 质量检查清单
S03
🔍
开源侦察官
Open Source Scout · 七维评估矩阵

七维度评估矩阵：功能性30%/易用性20%/性能15%/可维护性10%/社区活跃度10%/兼容性10%/文档5%，输出技术选型推荐。

七维矩阵 开源搜索 选型建议
INPUT技术需求说明 + 技术栈约束 OUTPUT评估矩阵表 + 推荐方案 + 备选方案 + 降级路径
S04
🗺️
执行规划官
Planner · P1-T2-S3三层拆解

将SOP工程包转化为P（阶段）/T（任务）/S（步骤）三层可执行操作手册，每步含工具/命令/预期结果/验证方式。

P/T/S三层 断点恢复 操作手册
INPUTS02 SOP文档 + S03选型报告 OUTPUT分阶段操作手册 + FAQ + 环境配置清单
S05
✅
测试验收工程师
Validator · 五维量化验收

五维度验收：功能完整性40%/文档完整性25%/可执行性20%/接口规范10%/安全5%。发布决策：PASS/CONDITIONAL_PASS/FAIL。

五维验收 P0缺陷检测 发布决策
INPUTS04操作手册 + 交付物清单 OUTPUT验收报告 + 缺陷列表 + PASS/FAIL决策 + 飞轮改进建议

五种执行模式

A — 单Skill

选择一个Skill独立运行，适合单一任务快速验证

B — 步进审批

每个Skill输出后暂停，用户审批后继续下一步

C — 全自动

六层Skill依次自动执行，无需人工干预

D — 圆桌+单跑

多视角圆桌审议后，再执行单次完整流水线

E — 深度精炼

多轮迭代执行，每轮基于上次输出持续优化

09  ·  ONBOARDING

引导流程

9步端到端用户引导，从准备环境到输出结果迭代，覆盖完整使用生命周期。

Step 1 — 准备环境

完成安装（参见第05章），确认 pnpm dev 无报错，浏览器访问 localhost:3000 看到首页

Step 2 — 浏览首页

首页展示项目列表、最近运行记录、快速统计。点击右上角"新建项目"按钮

Step 3 — 创建项目

填写项目名称（如"电商推荐系统"）、描述、选择默认AI模型（claude-3-5-sonnet推荐）

Step 4 — 选择执行模式

新手推荐模式B（步进审批），可以逐步了解每个Skill的输出。高级用户选模式C全自动

Step 5 — 输入需求

在执行控制台输入框填写需求，越详细越好。支持多段落。点击"开始执行"按钮

Step 6 — 观察执行过程

实时查看每个Skill的流式输出。模式B下每步完成后有"通过"/"重试"按钮，可干预流程

Step 7 — 查看结果报告

执行完成后跳转结果页，包含每层Skill的详细输出，可切换JSON/Markdown视图

Step 8 — 导出报告

点击"导出"按钮，选择下载JSON（用于程序处理）或Markdown（用于文档分享）

Step 9 — 迭代优化

基于首次输出，修改需求描述后再次运行。或切换到模式E深度精炼，持续迭代至满意

10  ·  CASE STUDY

模拟案例

两个完整端到端案例，展示每层Skill的真实输入输出，帮助理解系统工作方式。

场景： 某电商创业公司CTO，需要规划一套基于用户行为的个性化推荐系统，团队3人，预算有限。
S00

路由导航官 输出

INPUT

"我需要给我们电商平台做个推荐系统，基于用户浏览记录和购买历史，团队只有3个人，要快速上线"

OUTPUT（置信度 87%） 
intent: system_planning
confidence: 0.87
domain: e-commerce/recommendation
scale: small_team_3p
priority: speed_to_market
recommended_route: [S01, S02, S03, S04]
skip: S05 # 先快速验证，后期补验收
S02

SOP工程师 输出（摘要）

OUTPUT — 10阶段SOP
1 需求确认：定义推荐场景（首页/商品页/购物车），确定冷启动策略
2 数据架构：用户行为表Schema，事件埋点规范（click/view/purchase）
3 算法选型：协同过滤 vs 内容过滤 vs 混合推荐，小团队推荐轻量实现

12  ·  TIPS

使用技巧

12条实践技巧，分四组掌握系统的高阶用法，提升输出质量。

A — 输入优化

技巧 1：给定约束条件

在需求中明确团队规模、技术栈偏好、预算上限。S03选型会自动过滤不符合约束的方案。

技巧 2：描述背景而非解决方案

不要预设答案，描述问题背景。"我的用户流失率高"比"给我做一个用户留存系统"效果更好。

技巧 3：设定优先级

明确"最重要的一件事"，如"以快速上线为第一优先，可以牺牲扩展性"。S04规划会据此调整任务顺序。

B — 模式选择

技巧 4：新需求用模式B

第一次执行新类型需求，用步进审批模式观察每个Skill输出，建立对系统的信任感后再切换模式C。

技巧 5：重复任务用模式C

对于熟悉的任务类型，全自动模式C效率最高。结合模板化输入可实现批量处理。

技巧 6：重要决策用模式D

技术选型、架构设计等高影响决策，先圆桌审议避免视角盲区，再单次精确执行。

C — 执行策略

技巧 7：分阶段运行

大型项目不要一次跑全部Skill。先用模式A运行S00验证需求理解，再决定后续路径。

技巧 8：充分利用S05验收

在交付前必跑S05。即使前面Skill输出看起来很好，S05可能发现隐性缺口（P0缺陷立即阻止发布）。

技巧 9：保存中间结果

每次S01输出的优化Prompt值得单独保存。可复用于后续同类项目，无需重复S00-S01流程。

D — 输出利用

技巧 10：JSON结果接入工具链

S04输出的JSON任务列表可直接导入Jira/Linear/GitHub Issues。字段映射：title→task title, steps→subtasks。

技巧 11：Markdown用于文档

S02 SOP的Markdown输出可直接粘贴到Notion/Confluence，格式完整，无需二次编辑。

技巧 12：建立项目模板库

将同类项目的第一次执行结果存档，作为后续同类项目的输入参考。形成组织级Skill知识库。
